测试工具

cppunit：

  • 安装：

  • Fixture：一个或一组测试用例的测试对象。可以是你要测试的对象或者函数。
  • TestCase：测试用例。是对测试对象的某个功能或流程编写的测试代码。对一个Fixture，可能有多个测试用例。通过继承它来实现测试用例。继承自Test和TestFixture，
  • TestSuite：同时运行的测试用例的集合。可以是一个fixture的多个测试函数、也可以是多个fixture的所有测试用例。
  • TestFixture：测试固件。用来建立测试基准或构建测试用例的先决条件。
  • TestRunner：用来运行测试用例。接口为：virtual void addTest(Test *test)，virtual void run( TestResult &controller, const std::string &testPath = “” );TestRunner可以自行管理测试对象的生命周期，不用我们自己delete掉new的对象
  • TestListener：用来接收执行测试用例过程中产生的结果。
  • TestResult：用来管理TestListener。
  • TestCaller：TestCase适配器（Adapter），它将成员函数转换成测试用例。解决teatcase只能支持一个测试用例的问题，可以支持一组测试用例,如下
  • 使用时，在测试的主文件中将TestCase注册到TestSuite中，并运行，示例：
    Outputter：结果输出，可以定制不同输出格式
  • CppUnit定义一组宏用于检测错误，断言失败时，就会把错误信息报告给TestReslt
  • TestFactory：测试工厂，这是一个辅助类，通过借助一系列宏定义让测试用例的组织管理变得自动化
  1. googletest:将相关的测试分组为测试套件，可以共享数据和子例程；不会再第一个失败时停止，进停止当前测试而继续运行下一个
    • 编译安装
  • 断言：包括assert_*（产生的是致命失败，只有当失败以后执行没有意义dai’ma’ji’x才会使用，并且assert产生失败会立刻返回，可能会跳过清理代码，造成内存泄露），expect_*（产生的是不致命的失败，不终止当前函数，允许在一个测试报告中报告更多失败），注：任何可以被传进ostream东西都可以被传进断言的宏，如：

  • 测试：使用断言验证被测代码的行为，如果一个测试崩溃或有失败的断言，则失败
  • 测试套件：包含一个或多个测试。需要把测试分组到反应北侧代码结构的测试套件中。
  • 简单测试TEST(TestSuitName（测试套件的名字）,TestName（测试用例中测试的名字：测试套件名+自己的独立名字）):
  • 测试夹具TEST_F()：为多个不同的测试复用相同的对象配置，在使用之前必须先定义一个测试夹具类，继承自::testing::Test，编写默认构造函数SetUp()和析构函数。
  1. googlemock：模拟接口的工具
    • 定义Mock接口类：使用MOCK_METHOD后加一个数字，表明需要mock的函数有几个参数，MOCK_METHOD宏的第一个参数是函数名，第二个参数是函数指针类型。

  1. jmeter：用于接口测试和压力测试，基于JAVA，使用前需要先安装jdk。安装：

    • 创建线程组：线程组就是按照线程性质进行分组。一个进程=多个线程组，一个线程组=多个线程 
    • 设置线程数来模拟用户数量： 线程数：就是虚拟用户的数量，一个虚拟用户占用一个线程；准备时常：这些个虚拟用户要多长时间全部启动；循环次数：每个用户（线程）发送的请求次数；调度器：设置线程组启动的开始时间和结束时间；持续时间：测试持续的时间；启动延迟：测试延迟启动时间
    • sampler采样器： 向服务器发送请求，记录响应时间的最小单元和响应信息。如发送HTTP请求：名称：用于标识一个取样器；服务器名称或IP：HTTP请求发送的目标服务器名称或IP地址；端口号：目标服务器的端口号；方法：发送HTTP请求的方法；Content encoding：内容的编码方式 ；路径：目标URL路径（
    • 监听器：动态监控JMeter测试执行的结果，实时查看测试结果 。包括结果树和聚合报告。
      • 结果树：可以看到每个请求的结果，红色出错、绿色通过 ，Thread Name ：线程组名称；
      • 聚合报告： lable ：对应每一个 http 请求，显示的是 http 请求的 Name ，如百度 http 请求 name 为 baidu；
    • 断言：assert，用来判断响应的数据是否符合预期 。包含响应断言，json断言（采用json数据格式），xml断言
    • 定时器：定义请求之间的延迟间隔，就是隔多久请求一次

  2. QTP：quick test profession，用于回归测试。需要录制脚本，可以录制以后在代码化脚本修改
