（二）构造函数

发表于 2022-04-02 更新于 2022-07-11 分类于 C++多态与继承

包括默认构造函数、拷贝构造函数、列表初始化成员这几种构造函数，在什么情况下编译器产生的是有效的构造函数

默认构造函数

  • 当定义一个类的时候没有显示的为其声明构造函数，编译器会自动为其生成默认构造函数，但一般来说编译器生成的默认构造函数是没用的，只有在以下四种情况下才有用

含有（具有默认构造函数的）类对象成员

  • 即一个类没有显示的声明构造函数，但他含有一个类对象成员，该成员声明了构造函数，那么此时这个类的默认构造函数就是有用的。编译器会为这个类生成一个默认构造函数，但只有在构造函数真正被调用时才进行。如

    类Bar中包含了对象成员foo，而foo有显式声明的构造函数，这是编译器会为Bar生成默认构造函数，它调用Foo的构造函数来初始化对象成员foo，但不初始化Bar的str成员，此时需要我们自己初始化str

  • 当我们已经为str自己定义了一个构造函数，由于该构造函数里没有初始化对象成员，所以编译器还是会自动生成一个默认构造函数，首先调用成员对象的构造函数再加入我们自己定义的构造函数，即

继承自（带有默认构造函数）基类

  • 如果一个没有显示定义构造函数的类派生于一个有显示构造函数的基类，那么此时该派生类的默认构造函数会调用上一层基类的默认构造函数
  • 如果一个派生类声明了多个构造函数，但就是没有默认构造函数，那么编译器会扩展所有构造函数，将调用所有必要的默认构造函数（包括基类的，对象成员的）的代码加入这些构造函数之中。

Default Memberwise Initialization(默认逐成员初始化)

  • 当用户没有显示的定义拷贝构造函数，编译器内部以 default memberwise initialization 方法进行拷贝构造。该方法把类对象成员以外的成员变量的值，依次从某个对象拷贝到另一个对象身上。即：

  • 因此，决定一个编译器生成的是否是有用的拷贝构造函数，取决于这个类是否展现出了bitwise copy semantics（逐位次拷贝语义），如果没有展现出来那么生成的拷贝构造函数就是有用的。

bitwise copy semantics

  • 不会展现出bitwise copy semantics的情况
    • class包含一个对象成员，这个对象成员的类中声明了一个拷贝构造函数（可以是显示声明的，也可以是被编译器合成的）
    • class继承自有拷贝构造函数的基类
    • class声明了一个或多个虚函数
    • class继承自一个继承链，继承链中有一个或多个虚拟基类
  • 前两种中编译器将对象成员或基类的拷贝构造函数调用代码插入到生成的拷贝构造函数中

总结

  • 构造函数执行过程：
    • 在派生类构造函数中，所有虚基类和上一层基类的构造函数首先会按顺序被调用，以生成基类子对象
    • 上述完成后，对象的vptr会被初始化，指向正确的虚函数表
    • 如果有成员初始化列表，将在vptr设定后扩展，以免其中调用了虚函数
    • 如果有对象成员且尤其构造函数，调用对象成员的构造函数
    • 最后执行程序员提供的初始化代码
