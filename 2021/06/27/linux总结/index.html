linux总结

发表于 2021-06-27 更新于 2022-06-30 分类于 linux

常用命令

终端快捷键

  • crtl+a：移动到命令行开始
  • crtl+e：移动到命令行末尾
  • crtl+l： 清理命令行
  • crtl+c： 终止命令
  • !$： 显示系统最近用过的一条参数（比如说最近用过的一条路径）

gonome快捷键

  • alt+F1：同Windows的win，打开应用程序菜单
  • alt+F2：同Windows的win+R，打开命令
  • crtl+alt+d：返回桌面
  • crtl+alt+l：锁屏
  • alt+tab：切换程序窗口

文件查找

grep

  • grep -color “*” /路径 :在文件中高亮“*”
  • grep -v “*” :查找文件中不包含“*”的行
  • grep -n “*” :显示“*“在文件中的行号
  • grep -c “*”:显示有多少行”*“
  • grep -A 1 “*”：显示包含”*“的一行和其之后的一行
  • grep -B 1 “*”：显示包含行和之前的一行
  • grep -C 1”*”：显示包含行和前后一行
  • grep -i “*”：不区分大小写
  • grep -l “a”*：查找批量文件中包含a的文件
  • grep -L “a”*：查找批量文件中不包含a的文件
  • grep正则：可以识别^,$,.,[,],*，如查找以a开头 grep ‘^a’，以b结尾 grep ‘b$’
  • grep -w “a”:查找单词a

egrep

  • 适用于多条件查找，支持的正则表达式除了grep中的还包括（，），、，{，}，？，+，|等

fgrep

  • 不支持正则表达式，因此可以直接在文件中查找正则表达式里面的符号

vim

  • 一般指令模式：默认模式，只读
  • 编辑模式：+”i“，编辑文本
  • 指令列模式：+”：“，用于保存退出

磁盘

  • SATA：抗干扰性强，支持热插拔
  • SCSI：多用于个人工作站，高转速，传输时CPU占用率低
  • SAS：采取序列式技术，更高的传输速度

分区

  • MBR：第一个扇区最重要，包含446bytes的主要开机记录和64bytes的分区表，分区表最多分为四个分区，一个主分区和扩展分区
  • GPT：针对不同磁盘有不同大小的扇区，在定义扇区上使用逻辑区块地址，默认大小512bytes。第一个区块记录主要开机记录，后33个区块记录分区信息，除了第一个分区做表头，剩下的32个分区最多可以分为32*4=128个分区，且都是主分区

开机检测程序BIOS

  • basic input/output system，基本输入/输出系统，存放在断电后不会丢失的只读内存中。
  • 开机后执行的第一个程序，知道可以开机的磁盘并读取磁盘第一个扇区的主要开机记录，再由主要开机记录执行其中的开机程序
  • 主要开机记录：包含选单、载入核心文件、转交其他开机管理程序。转交功能可以实现多重引导

文件系统

  • inode：一个文件占用一个inode（128bytes），记录文件权限、所有者、容量、更改时间等，以及记录文件内容所在的block编号，可以使用直接引用和间接引用
  • block：记录文件的内容
  • 目录：创建目录时，会分配一个inode和至少一个block,block记录的时所有文件的inode编号和文件名

文件与目录的基本操作

  • ls -a：列出全部文件/-d:列出目录/-l:以长数据串行列出，包含文件属性、权限
  • cd：更改目录
  • mkdir：创建目录 -m：配置目录权限/-p：递归创建目录
  • rmkdir：删除目录
  • touch：更新文件时间
  • cp：复制文件 -i：若目标文件已存在就要先询问/-u：只有目标不存在或者比较旧才更新
  • rm：删除文件
  • mv:移动文件
  • 修改权限：chmod [u/g/o/a] [+-=] [rwx] dirname/filename
  • 文件默认权限：没有可执行权限（666）
  • 目录默认权限：必须有可执行权限（777）

获取文件内容

  • cat：获取文件内容
  • tac：从该文件的最后一行开始打印
  • more：适合大文件查看，可以一页一页查看
  • less：多了一个向前翻页
  • head - n：得到文件的前n行
  • tail -n：得到文件的后几行
  • od：以字符或16进制显示二进制文件

指令与文件搜索

  • 搜索指令：which [-a] commond
  • 文件搜索：whereis [-b/m/s/u] dirname/filename；locate [-i/r] keyword；find -name “*“

文件压缩与打包

  • gzip -c：将压缩数据输出到屏幕上/-d：解压缩/-t：显示压缩文件是否出错/-v：显示压缩比等信息
  • bzip2 -k：保留原文件压缩
  • xz -dtlkc# filename：最佳的压缩比
  • 打包压缩：tar -jcv -f filename.tar.bz2
  • 查看：tar -jtv -f filename.tar.bz2
  • 解压缩：tar -jxv -f filename.tar.bc2 -C要解压缩的目录

提取指令

  • cut -d ：后接分隔符
  • cut -f n ：在分隔符分割以后，取第n个区间
  • cut -c n- ：以字符为单位取出区间

排序指令

  • sort -f：忽略大小写
  • sort -b ：忽略前面空格
  • sort -r ：反向排序
  • sort -u ：重复内容只出现一次
  • sort -t ：以“*”为分隔符
  • sort -k ：指定区间的排序
  • uniq -i：重复数据只取一个，不区分大小写
  • uniq -c ：重复数据只取一个并计数

双向输出重定向

  • tee [-a] file ：不仅在屏幕上显示输出，还会传送到文件中

字符转换指令

  • tr [-d] ：-d用来删除一行，否则用来替换字符
  • col ：将tab转成空格
  • expand -t ：将tab转成t个空格
  • join ：将相同数据的两行合并
  • paste ：将两行粘贴在一起
  • split [-bl] file PREFIX：将一个文件划分成多个文件，-b以大小分区，-l以行数分区

awk

  • awk $n ：每次处理一行，以字段为单位进行处理，n是字段号，$0表示一整行
  • awk ‘条件类型1 {动作1} 条件类型2 {动作2}…’ filename：根据字段的条件进行匹配 FS:分隔符 NF：每一行拥有的总字段数 NR：目前处理到第几行

进程

  • ps -l：查看自己的进程； aux：查看系统进程；ps aux | grep thread：查看特定进程
  • netstat -anp | grep port ：查看占用端口的进程
  • wait()：收到子进程退出的信号，销毁子进程
  • 孤儿进程：子进程还在运行而父进程退出了，此时孤儿进程会被init进程收养，不会对系统造成危害
  • 僵尸进程：子进程已经退出，但父进程一直不调用wait或waitpid，导致子进程的进程描述一直在系统中占用进程号，成为僵尸进程。此时需要将他们的父进程杀死，然后僵尸进程会变成孤儿进程被init进程收养，释放掉资源
